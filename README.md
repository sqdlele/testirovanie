# Репозиторий заданий

Один репозиторий: **каждая папка — одно задание**. Нумерация папок обязательна (01, 02, 03, …).

## Структура

```
zadaniya/
├── 01-calculator/     # Задание 1. Калькулятор
├── 02-megachat/       # Задание 2. Чат
├── 03-.../            # Задание 3. ...
└── README.md
```

В каждой папке — только **ваши файлы с решением** (исходный код и тесты). Остальное в репозиторий не загружаем.

## Что загружать

- **Нужно:** ваш код решения и тесты к нему.  
  Например: класс калькулятора и тесты к калькулятору; код чата и тесты приложения.
- **Не нужно:** библиотеки, виртуальные окружения (`venv`, `node_modules`), настройки IDE (`.idea`, `.vscode`), личные файлы (фото и т.п.), служебные скрипты и конфиги, которые не относятся к заданию.

## Именование папок

- Номер с нулём: `01`, `02`, `03`, … (чтобы порядок сохранялся).
- Можно добавить короткое название: `01-calculator`, `02-megachat`.

В папке задания может быть подпапка с проектом (например, `02/megachat/`). В каждой папке задания можно положить свой `README.md` с описанием задания и инструкцией по запуску и тестам.

## Как залить на Git

Репозиторий должен быть **один** — в корне папки со всеми заданиями (там, где лежит этот README). Не создавайте отдельный репозиторий в каждой папке задания.

### 1. Один раз: инициализация и первый коммит

В корне папки (например, `d:\zadaniya`) откройте терминал и выполните:

```bash
git init
git add .
git status
```

Проверьте по `git status`, что в список попали только нужные файлы (папки заданий, README, .gitignore). Не должно быть `venv/`, `node_modules/`, `.idea/` и т.п. — их исключает `.gitignore`.

Затем:

```bash
git commit -m "Первая загрузка: структура репозитория и задания"
```

### 2. Подключить удалённый репозиторий

Создайте **пустой** репозиторий на GitHub (или GitLab, Bitbucket) — без README и без .gitignore.

Скопируйте URL репозитория (HTTPS или SSH) и выполните (подставьте свой URL):

```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПОЗИТОРИЙ.git
```

Проверка:

```bash
git remote -v
```

### 3. Отправить код на сервер

Обычно основная ветка называется `main`:

```bash
git branch -M main
git push -u origin main
```

Введите логин и пароль (или токен) от GitHub, если попросит. После этого код будет на GitHub.

### 4. Дальнейшая работа

Когда добавляете новое задание или меняете код:

```bash
git add .
git status
git commit -m "Задание 3: описание"
git push
```

**Важно:** если внутри папки задания (например, `02/megachat`) уже есть своя папка `.git`, получится вложенный репозиторий — в общий репозиторий эта папка попадёт как одна ссылка, а не как файлы. Чтобы всё было в одном репо, удалите внутреннюю `.git`: зайдите в `02/megachat`, выполните `rmdir /s /q .git` (Windows) или `rm -rf .git` (Linux/macOS), затем снова `git add .` и `git commit` в корне.
